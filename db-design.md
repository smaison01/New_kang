# DB 설계

## 현재 환경
- **DB**: H2 인메모리 (개발용)
- **ORM**: Spring Data JPA + Hibernate
- **DDL 전략**: `create-drop` (앱 시작 시 테이블 생성, 종료 시 삭제)

---

## 테이블 구조

### post

| 컬럼명 | 타입 | 제약조건 | 설명 |
|--------|------|----------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 게시글 번호 |
| title | VARCHAR(255) | NOT NULL | 제목 |
| content | TEXT | | 내용 |
| author | VARCHAR(255) | | 작성자 |
| created_at | TIMESTAMP | | 작성일시 (저장 시 자동 입력) |

```sql
CREATE TABLE post (
    id         BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    title      VARCHAR(255) NOT NULL,
    content    TEXT,
    author     VARCHAR(255),
    created_at TIMESTAMP(6)
);
```

---

## H2 콘솔 접속 (개발 중 DB 직접 확인)

1. 백엔드 실행 중 브라우저에서 접속: `http://localhost:8080/h2-console`
2. 설정값:
   - JDBC URL: `jdbc:h2:mem:boarddb`
   - User Name: `sa`
   - Password: (없음)

---

## 주의사항

- 백엔드 재시작 시 **데이터 전부 초기화됨** (`create-drop` 설정 때문)
- 데이터를 유지하려면 아래 옵션 중 하나 선택:

### 옵션 1 — H2 파일 모드로 전환 (간단)
`application.properties` 수정:
```properties
spring.datasource.url=jdbc:h2:file:./data/boarddb
spring.jpa.hibernate.ddl-auto=update
```

### 옵션 2 — MySQL로 전환
`pom.xml` 의존성 교체:
```xml
<dependency>
    <groupId>com.mysql</groupId>
    <artifactId>mysql-connector-j</artifactId>
</dependency>
```
`application.properties` 수정:
```properties
spring.datasource.url=jdbc:mysql://localhost:3306/boarddb
spring.datasource.username=root
spring.datasource.password=비밀번호
spring.jpa.hibernate.ddl-auto=update
```
